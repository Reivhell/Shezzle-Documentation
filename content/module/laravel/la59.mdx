## Tujuan Pembelajaran

Menguasai Laravel Pint untuk menjaga konsistensi code style dengan otomatisasi fixing, konfigurasi custom, dan integrasi dalam development workflow.

## 1. Instalasi & Penggunaan Dasar

### 1.1 Instalasi Pint

```bash
# Pint sudah include di Laravel 10+ / fresh installs
# Untuk existing projects atau global install:
composer require laravel/pint --dev

# Atau global install
composer global require laravel/pint
```

### 1.2 Basic Commands

```bash
# Check code style issues (dry-run)
./vendor/bin/pint

# Fix code style issues
./vendor/bin/pint --fix

# Atau tanpa --fix akan otomatis fix juga
./vendor/bin/pint

# Show progress
./vendor/bin/pint --test

# Specific files/directories
./vendor/bin/pint app/Models
./vendor/bin/pint app/Http/Controllers/UserController.php

# Show diff
./vendor/bin/pint --diff
```

## 2. Presets

### 2.1 Available Presets

```bash
# Laravel preset (default)
./vendor/bin/pint --preset=laravel

# PER (PHP Evolving Recommendations)
./vendor/bin/pint --preset=per

# PSR-12
./vendor/bin/pint --preset=psr12

# Symfony
./vendor/bin/pint --preset=symfony
```

### 2.2 Preset Configuration

```json
// pint.json
{
    "preset": "laravel"
}
```

```json
// Atau dengan custom rules
{
    "preset": "psr12",
    "rules": {
        "ordered_imports": true,
        "no_unused_imports": true
    }
}
```

## 4. CI/CD Integration

### 4.1 GitHub Actions

```yaml
# .github/workflows/pint.yml
name: Code Style

on: [push, pull_request]

jobs:
  pint:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup PHP
      uses: shivammathur/setup-php@v2
      with:
        php-version: '8.2'
    
    - name: Install Dependencies
      run: composer install --no-interaction --prefer-dist
    
    - name: Run Pint
      run: ./vendor/bin/pint --test
    
    - name: Fix Code Style (jika failure dan auto-fix enabled)
      if: failure()
      run: |
        ./vendor/bin/pint
        git config --local user.email "action@github.com"
        git config --local user.name "GitHub Action"
        git add .
        git commit -m "Apply code style fixes" || exit 0
        git push
```

### 4.2 Pre-commit Hooks

```bash
# .git/hooks/pre-commit
#!/bin/bash

files=$(git diff --cached --name-only --diff-filter=ACM | grep '\.php$')

if [ -n "$files" ]; then
    ./vendor/bin/pint $files --test
    
    if [ $? -ne 0 ]; then
        echo "Code style issues found. Run './vendor/bin/pint' to fix."
        exit 1
    fi
fi
```

```bash
# Make executable
chmod +x .git/hooks/pre-commit
```

### 4.3 Husky + lint-staged (Node.js projects)

```json
// package.json
{
  "lint-staged": {
    "*.php": "./vendor/bin/pint --test"
  },
  "husky": {
    "hooks": {
      "pre-commit": "lint-staged"
    }
  }
}
```

## 5. Parallel Processing

```bash
# Pint menggunakan parallel processing secara default
# Untuk disable (jika ada issues):
./vendor/bin/pint --no-parallel

# Atau di konfigurasi
{
    "preset": "laravel",
    "parallel": false
}
```

## 6. Output Formats

```bash
# Default (colored CLI)
./vendor/bin/pint

# JSON (untuk parsing)
./vendor/bin/pint --format=json

# GitLab Code Quality report
./vendor/bin/pint --format=gitlab

# Checkstyle XML
./vendor/bin/pint --format=checkstyle

# JUnit XML
./vendor/bin/pint --format=junit
```

## Kesimpulan

Laravel Pint menyederhanakan code style consistency:
- **Zero configuration** dengan sensible defaults
- **Multiple presets** (Laravel, PER, PSR-12, Symfony)
- **Extensive rules** untuk customization
- **CI/CD integration** untuk automated checking
- **Parallel processing** untuk performa

Selanjutnya di **Sesi 60**, kita akan mempelajari Laravel Sail (Docker Development).
