# Sesi 1: Pendahuluan & Konsep Inti Astro Framework

## 1.1 Apa itu Astro Framework?

**Astro** adalah framework web modern yang dirancang untuk membangun website yang **cepat, ringan, dan berfokus pada konten**. Bayangkan Astro seperti seorang arsitek yang sangat memperhatikan efisiensi: dia tidak akan membangun gedung pencakar langit hanya untuk sebuah warung kopi kecil. Begitu juga Astro‚Äîia hanya akan mengirimkan JavaScript yang benar-benar dibutuhkan ke browser pengguna.

<InfoBox title="üí° Analogi Sederhana" variant="tip" icon="lightbulb">
Bayangkan kamu mengirim paket ke teman. Framework tradisional seperti mengirim seluruh gudang beserta isinya, padahal temanmu hanya butuh satu barang kecil. Astro? Ia hanya mengirim barang yang diminta, ditambah sedikit "manual penggunaan" jika diperlukan. Hasilnya: paket lebih ringan, lebih cepat sampai, dan temanmu tidak perlu repot memilah-milah barang yang tidak dibutuhkan.
</InfoBox>

### Sejarah Singkat:
- **2021**: Astro diluncurkan oleh Fred K. Schott dan tim di Snowpack
- **2022**: Memperkenalkan Islands Architecture yang revolusioner
- **2023-2024**: Content Collections dan View Transitions
- **2025**: Server Islands dan Astro Actions
- **2026**: Fitur-fitur stabil seperti Live Content Collections dan peningkatan performa besar-besaran

## 1.2 Filosofi & Konsep Inti

Astro dibangun di atas **tiga pilar filosofis** yang menjadi DNA-nya:

### 1. **Content-First (Konten di Utamakan)**
Astro dirancang untuk website yang berfokus pada konten: blog, dokumentasi, portofolio, e-commerce, dan situs marketing. Bukan untuk aplikasi web berat seperti dashboard analitik real-time atau game online.

<InfoBox title="Contoh Perbandingan" variant="info" icon="compare">
**‚ùå Tidak Cocok untuk Astro:** Gmail, Google Docs, Figma (web app interaktif berat)

**‚úÖ Sangat Cocok untuk Astro:** Blog, Dokumentasi, Landing Page, E-commerce, Portofolio
</InfoBox>

### 2. **Zero JavaScript by Default (Nol JavaScript Secara Default)**
Ini adalah keunikan Astro. Secara default, Astro mengirimkan **HTML murni** tanpa JavaScript. JavaScript hanya dikirim jika kamu secara eksplisit memintanya.

<InfoBox title="üçΩÔ∏è Analogi" variant="tip" icon="restaurant">
Seperti restoran yang tidak otomatis menyertakan saus sambal di setiap pesanan. Jika pelanggan butuh, mereka bisa meminta. Jika tidak, makanan tetap enak dinikmati tanpa tambahan apa pun.
</InfoBox>

### 3. **Ship Less, Do More (Kirim Lebih Sedikit, Lakukan Lebih Banyak)**
Astro mengoptimalkan otomatis: gambar, font, CSS, dan JavaScript. Tujuannya satu: **Core Web Vitals** yang sempurna.

## 1.3 Islands Architecture (Arsitektur Pulau)

Ini adalah **teknologi inti** Astro yang paling revolusioner.

### Analogi: Restoran All-You-Can-Eat vs Restoran √† la Carte

**Framework Tradisional (React/Vue/Angular) = Restoran All-You-Can-Eat**
- Kamu bayar satu harga, dapat semua makanan (JavaScript)
- Padahal mungkin kamu hanya mau salad dan air putih
- Hasilnya: banyak makanan terbuang (JavaScript tidak terpakai di-download)

**Astro Islands = Restoran √† la Carte**
- Kamu pesan hanya apa yang kamu makan
- Setiap "pulau" interaktivitas adalah pesanan terpisah
- Hasilnya: tidak ada pemborosan, bayar hanya untuk yang dimakan

### Definisi Teknis:

**Islands Architecture** adalah teknik di mana halaman web dirender sebagai HTML statis, kemudian "pulau-pulau" kecil JavaScript dihidupkan secara terpisah di tempat yang membutuhkan interaktivitas.

<CodeBlock filename="example.astro" language="astro" code={`---
// Ini adalah "Frontmatter" - kode server-side
import ImageCarousel from '../components/ImageCarousel.jsx';
import CommentSection from '../components/Comments.vue';
---

<!-- HTML Statis - Zero JavaScript -->
<header>
  <h1>Blog Saya</h1>
</header>

<main>
  <article>
    <p>Ini adalah konten artikel dalam HTML murni...</p>
  </article>
  
  <!-- "Pulau" 1: React component, hanya aktif saat terlihat -->
  <ImageCarousel client:visible />
  
  <!-- "Pulau" 2: Vue component, aktif saat browser idle -->
  <CommentSection client:idle />
</main>

<footer>
  <p>&copy; 2026 Blog Saya</p>
</footer>`} />

### Fungsi Direktif `client:*`:

| Direktif | Fungsi | Use Case |
|----------|--------|----------|
| `client:load` | Langsung load JavaScript saat halaman load | Komponen penting yang harus langsung interaktif |
| `client:idle` | Load saat browser tidak sibuk | Komponen sekunder, tidak urgent |
| `client:visible` | Load saat komponen masuk viewport | Komponen di bawah fold (lazy load) |
| `client:media` | Load berdasarkan media query | Komponen mobile/desktop spesifik |
| `client:only` | Tidak render di server, hanya client | Komponen yang butuh browser API |

## 1.4 Keunggulan Astro vs Framework Lain

Mari kita bandingkan secara objektif:

### Perbandingan Performa (Real-world Data):

| Metrik | Astro | Next.js | Gatsby | Nuxt | Plain React |
|--------|-------|---------|--------|------|-------------|
| **JavaScript yang Dikirim** | 0 KB (default) | 70-200 KB | 100-300 KB | 80-200 KB | 150-400 KB |
| **Time to Interactive** | Sangat Cepat | Cepat | Sedang | Cepat | Lambat |
| **First Contentful Paint** | Excellent | Good | Good | Good | Fair |
| **Build Time** | Sangat Cepat | Cepat | Lambat | Cepat | N/A |
| **Kompleksitas** | Rendah | Sedang | Sedang | Sedang | Tinggi |

### Use Case Spesifik:

<InfoBox title="‚úÖ Pilih Astro jika:" variant="tip" icon="check_circle">
- Membangun blog, dokumentasi, atau situs konten
- Prioritas utama adalah kecepatan dan SEO
- Ingin fleksibilitas menggunakan React, Vue, Svelte, dll dalam satu project
- Tidak ingin terikat pada satu ekosistem framework
</InfoBox>

<InfoBox title="‚ö†Ô∏è Pilih Next.js/Nuxt jika:" variant="warning" icon="warning">
- Membangun aplikasi web kompleks dengan banyak state management
- Butuh fitur real-time yang intensif
- Aplikasi dashboard atau admin panel yang sangat interaktif
</InfoBox>

### Contoh Kode Perbandingan:

**Next.js (mengirim JavaScript untuk seluruh halaman):**

<CodeBlock filename="pages/index.js" language="jsx" code={`import { useState } from 'react';

export default function Home() {
  const [count, setCount] = useState(0); // JavaScript harus di-load
  
  return (
    <div>
      <h1>Selamat Datang</h1>
      <p>Ini halaman sederhana</p>
      <button onClick={() => setCount(count + 1)}>
        Klik: {count}
      </button>
    </div>
  );
}
// ‚ùå Seluruh halaman memerlukan React bundle (~70KB)`} />

**Astro (hanya mengirim JavaScript untuk tombol):**

<CodeBlock filename="pages/index.astro" language="astro" code={`---
// pages/index.astro
const title = "Selamat Datang"; // Server-side, tidak ada JS di client
---

<h1>{title}</h1>
<p>Ini halaman sederhana</p>

<!-- Hanya bagian ini yang memerlukan JavaScript -->
<Counter client:load />

<script>
  // Counter adalah komponen terpisah
  // JavaScript hanya di-load untuk komponen ini
</script>
// ‚úÖ Hanya ~2KB JavaScript untuk komponen Counter`} />

## 1.5 Use Cases & Kapan Menggunakan Astro

### 5 Use Case Terbaik untuk Astro:

#### 1. **Blog & Personal Website**

<CodeBlock filename="src/pages/blog/[slug].astro" language="astro" code={`---
// src/pages/blog/[slug].astro
import { getCollection } from 'astro:content';

export async function getStaticPaths() {
  const posts = await getCollection('blog');
  return posts.map(post => ({
    params: { slug: post.slug },
    props: { post },
  }));
}

const { post } = Astro.props;
const { Content } = await post.render();
---

<article>
  <h1>{post.data.title}</h1>
  <Content /> <!-- Render Markdown ke HTML -->
</article>`} />

**Kenapa Astro?** Zero JavaScript default, SEO sempurna, rendering Markdown native.

#### 2. **Dokumentasi (Docs Site)**

Astro memiliki **Starlight**, framework dokumentasi built-in di atas Astro.

<CodeBlock filename="terminal" language="bash" code={`npm create astro@latest -- --template starlight`} />

**Kenapa Astro?** Navigasi cepat, pencarian built-in, i18n support, dan performa luar biasa.

#### 3. **E-commerce (Product Showcase)**

<CodeBlock filename="products.astro" language="astro" code={`---
// Mengambil data produk dari CMS atau API
const products = await fetch('https://api.store.com/products').then(r => r.json());
---

<div class="product-grid">
  {products.map(product => (
    <ProductCard 
      title={product.name}
      price={product.price}
      image={product.image}
      client:visible <!-- Hanya interaktif saat terlihat -->
    />
  ))}
</div>`} />

**Kenapa Astro?** Gambar dioptimasi otomatis, loading cepat, integrasi dengan payment gateway via API routes.

#### 4. **Landing Page & Marketing Site**

<CodeBlock filename="landing.astro" language="astro" code={`---
// Zero JavaScript untuk konten statis
import Hero from '../components/Hero.astro';
import Features from '../components/Features.astro';
import Pricing from '../components/Pricing.astro';
---

<Hero />        <!-- HTML statis -->
<Features />    <!-- HTML statis -->
<Pricing />     <!-- HTML statis -->
<Newsletter client:idle /> <!-- Hanya form newsletter yang interaktif -->`} />

**Kenapa Astro?** Core Web Vitals sempurna, konversi lebih tinggi karena kecepatan.

#### 5. **Multi-Framework Project**

<CodeBlock filename="multi-framework.astro" language="astro" code={`---
// Kamu bisa mix React, Vue, Svelte dalam satu file!
import ReactHeader from '../components/Header.jsx';
import VueSidebar from '../components/Sidebar.vue';
import SvelteCounter from '../components/Counter.svelte';
---

<ReactHeader client:load />
<main>
  <VueSidebar client:visible />
  <SvelteCounter client:idle />
</main>`} />

**Kenapa Astro?** Tidak perlu rewrite, tim bisa menggunakan framework yang mereka kuasai.

### Kapan TIDAK Menggunakan Astro:

| Situasi | Alternatif yang Lebih Baik | Alasan |
|---------|---------------------------|--------|
| **Real-time App** (Chat, Game online) | Next.js, SvelteKit | Butuh WebSocket dan state management kompleks |
| **Dashboard Admin Berat** | React + Vite, Next.js | Banyak interaktivitas, sedikit konten statis |
| **Single Page Application (SPA)** murni | React, Vue, Angular | Astro optimasi untuk Multi-Page App (MPA) |
| **Aplikasi Mobile** | React Native, Flutter | Astro untuk web, bukan native mobile |
