## 7.1 Layout Components

Layout adalah kerangka/wrapper yang membungkus konten halaman. Mereka menyediakan struktur konsisten (header, footer, navigation) dan metadata yang reusable di seluruh website.

### Analogi Sederhana:

> Bayangkan layout seperti **template surat resmi perusahaan**. Setiap surat memiliki kop yang sama, alamat yang sama, dan footer yang sama. Hanya isi suratnya yang berbeda. Begitu juga layout Astro - struktur HTML yang sama, konten yang berbeda-beda.

### Struktur Dasar Layout:

## 7.2 Nested Layouts

Layout bisa bersarang (nested) - layout di dalam layout. Ini memungkinkan hierarki template yang kompleks.

### Struktur Nested Layouts:

```
src/layouts/
├── BaseLayout.astro       # Root layout (HTML, head, body)
├── BlogLayout.astro       # Layout khusus blog (extends BaseLayout)
├── ArticleLayout.astro    # Layout artikel (extends BlogLayout)
└── DocsLayout.astro       # Layout dokumentasi (extends BaseLayout)
```

## 7.4 Conditional Layouts

Mengganti layout berdasarkan kondisi tertentu (user role, device, preferences).

### Layout Berdasarkan User:

```astro
---
// src/layouts/ConditionalLayout.astro
import BaseLayout from './BaseLayout.astro';
import AdminLayout from './AdminLayout.astro';
import DashboardLayout from './DashboardLayout.astro';

interface Props {
  requiresAuth?: boolean;
  requiredRole?: 'user' | 'admin';
}

const { title, requiresAuth = false, requiredRole = 'user' } = Astro.props;

// Cek auth dari cookies atau session
const authToken = Astro.cookies.get('auth')?.value;
const userRole = Astro.cookies.get('role')?.value || 'guest';

// Redirect jika tidak auth
if (requiresAuth && !authToken) {
  return Astro.redirect('/login?redirect=' + encodeURIComponent(Astro.url.pathname));
}

// Pilih layout berdasarkan role
let LayoutComponent = BaseLayout;
if (userRole === 'admin') {
  LayoutComponent = AdminLayout;
} else if (authToken) {
  LayoutComponent = DashboardLayout;
}
---

<LayoutComponent title={title} userRole={userRole}>
  <slot />
</LayoutComponent>
```

### Layout Berdasarkan Route Pattern:

```astro
---
// src/layouts/RouteAwareLayout.astro
import MarketingLayout from './MarketingLayout.astro';
import AppLayout from './AppLayout.astro';
import PlainLayout from './PlainLayout.astro';

const { pathname } = Astro.url;

// Marketing pages: landing, about, pricing
const isMarketing = ['/', '/about', '/pricing', '/contact'].includes(pathname) ||
  pathname.startsWith('/blog');

// App pages: dashboard, settings, profile
const isApp = pathname.startsWith('/app') || pathname.startsWith('/dashboard');

const LayoutComponent = isMarketing 
  ? MarketingLayout 
  : isApp 
    ? AppLayout 
    : PlainLayout;
---

<LayoutComponent {...Astro.props}>
  <slot />
  <slot name="head" slot="head" />
</LayoutComponent>
```

```astro
</ComplexLayout>
```

### Slot Fallbacks:

```astro
const hasSidebar = Astro.slots.has('sidebar');

    </aside>

    {hasToc && (
      <nav class="toc">
        <slot name="toc" />
      </nav>
    )}
    
    <article>
      <slot />
    </article>
  </main>
</div>
```

## Ringkasan Sesi 7

| Konsep | Deskripsi | Use Case |
|--------|-----------|----------|
| **Base Layout** | Wrapper HTML dasar | Semua halaman |
| **Nested Layouts** | Layout di dalam layout | Hierarki kompleks (Blog → Article) |
| **Default Layouts** | Layout otomatis | Markdown/MDX files |
| **Conditional Layouts** | Layout berdasarkan kondisi | Auth, role, route pattern |
| **Named Slots** | Multiple content areas | Sidebar, header, footer terpisah |
| **Slot Fallbacks** | Default content jika slot kosong | Navigation default, empty states |

### Best Practices:

1. **DRY Principle**: Jangan duplikat HTML di setiap halaman
2. **Composition**: Gunakan nested layouts untuk fleksibilitas
3. **Type Safety**: Definisikan interface untuk layout props
4. **Performance**: Layout Astro = zero JS, gunakan framework components hanya jika perlu
5. **Accessibility**: Pastikan skip links, landmarks, dan heading hierarchy konsisten
